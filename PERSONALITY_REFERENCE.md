## 📝 Usage Patterns

### At Start of Query:
```
"can you act like [NAME] while [ACTION]"
"act like [NAME] and [ACTION]"
"pretend to be [NAME] and [ACTION]"
"as [NAME], [ACTION]"
"while acting as [NAME] [ACTION]"
```

### At End of Query:
```
"[ACTION] as [NAME]"
"[ACTION] while acting as [NAME]"
"[ACTION] like [NAME] would"
```

## 🔧 Technical Details

### Backend (Python)
- **File**: `llm/static_remote_LLM/apillm.py`
- **Function**: `extract_personality_from_query()`
- **Personalities**: 70+ in `special_cases` dict
- **Temperature**: 0.9 (more expressive)

### Web Frontend
- **File**: `web/src/features/pages/LLMChatPage.tsx`
- **All 6 endpoints** pass `query` parameter

### Mobile (Flutter)
- **File**: `mobile/lib/utils/personality_extractor.dart`
- **Class**: `PersonalityExtractor`
- **All 6 endpoints** pass `query` parameter

### How It Works
1. Query is analyzed for personality patterns
2. Personality extracted and capitalized properly
3. LLM prompt enhanced with character instructions
4. Temperature increased to 0.9 for creativity
5. Response maintains data accuracy with personality voice
